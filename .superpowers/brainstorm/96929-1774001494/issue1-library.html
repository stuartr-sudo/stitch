Issue #1: Library & Video Thumbnails

Icons too small, slow to load, video library shows no thumbnails

Current Problems

🔍 Icons Too Small

Type badges (image/video/audio) are tiny 14px icons. Hard to distinguish media types at a glance.

🐌 Slow Loading

Full-res images load via loading="lazy" but no thumbnail/placeholder system. Videos load full files.

📹 No Video Thumbnails

Video Studio library shows dark cards with "LIBRARY" text. No poster frames or thumbnails are being generated/stored.

Proposed Fixes

A

Thumbnail Generation Pipeline

When saving media to library, generate a 400px thumbnail (images) or extract first frame (videos) and store a thumbnail_url column. Show thumbnails in grids, full-res on click/hover.

Pros

  • Dramatically faster grid loading
  • Videos finally show previews
  • Lower bandwidth usage

Cons

  • Needs DB migration (add thumbnail_url columns)
  • Need to backfill existing items
  • Storage cost for thumbnails
B

Supabase Image Transforms (Quick Win)

If images are in Supabase Storage, use their built-in transform URL params (?width=400&height=300) for thumbnails. For videos, extract poster frame client-side on first load and cache.

Pros

  • No DB migration needed
  • Works with existing URLs
  • Instant for images

Cons

  • Only works for Supabase-hosted files
  • Client-side video frame extraction is janky
  • External URLs won't benefit
C

Hybrid: Pipeline + Transforms + Blurhash

Generate thumbnails on save (pipeline), use Supabase transforms as fallback, and add BlurHash placeholders for instant perceived loading. Store blurhash string per item.

Pros

  • Best UX — instant blurhash → thumbnail → full-res
  • Works for all URL types
  • Progressive loading feels fast

Cons

  • Most implementation work
  • BlurHash library dependency
  • Still needs DB columns + backfill

Icon Size Fix (Will Do Regardless)

✅ Simple CSS change: Increase type badges from w-3.5 h-3.5 → w-5 h-5 and badge container from p-1.5 → p-2. Also increase the grid to show larger cards on desktop (3 cols instead of 4 for better visibility).

Video Studio Specific Fix

📹 Video Studio library currently queries generated_videos but the table has no thumbnail column. The cards show "LIBRARY" text because there's no poster frame. Whichever approach we pick above will fix this — the key is storing/generating a thumbnail per video.
